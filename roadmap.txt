# Stage 4: Data Handling & Small Utilities ‚Äî Exercise Roadmap

## 1. Data Storage & Serialization‚úÖ

Exercise 1: Save API data to JSON
- Fetch data from any API (or use Stage 3 API results).
- Save selected fields to a JSON file.
- Include a timestamp and handle file write errors.

Exercise 2: Load JSON and parse data‚úÖ
- Read a JSON file.
- Extract certain fields (e.g., names, IDs).
- Handle missing fields and invalid JSON.

Exercise 3: Save data to CSV‚úÖ
- Convert API or JSON data into CSV.
- Include headers, handle special characters.
- Handle errors if file cannot be written.

## 2. Data Cleaning & Transformation‚úÖ

Exercise 4: Filter & transform data‚úÖ
- Read JSON or CSV.
- Remove invalid entries (e.g., missing fields, empty strings).
- Transform fields (e.g., uppercase names, calculate string lengths).

Exercise 5: Aggregation and summary‚úÖ
- Group data by a field (e.g., location or category).
- Compute totals, averages, or counts.
- Save results to a new JSON or CSV file.

Exercise 6: Combine multiple sources
- Merge two or more JSON or CSV files.
- Deduplicate entries.
- Handle inconsistent keys or missing fields.

## 3. Small Utility Scripts

Exercise 7: CLI Data Query Tool
- Use `argparse` or `click`.
- Accept filters like ‚Äúlocation‚Äù or ‚Äúmin quantity.‚Äù
- Print matching records from JSON or CSV.

Exercise 8: Batch Data Updater
- Read data from JSON or CSV.
- Update certain fields based on rules (e.g., increment quantities, change categories).
- Save updated data back to the file safely.

Exercise 9: Automated Data Reporter
- Generate a summary report (JSON, CSV, or TXT) from collected data.
- Include stats like counts, averages, or top items.
- Add timestamp and save in a separate reports folder.

## 4. Logging & Error Handling

Exercise 10: Add logging to your scripts
- Track start/end time, number of items processed, and errors.
- Log to a file with timestamps.

Exercise 11: Exception handling & retries
- Wrap API or file operations in try/except.
- Retry failed requests or operations a few times.
- Log any failures.

Exercise 12: Config & environment variables
- Move file paths, API URLs, or filter criteria into a `.env` or `config.json`.
- Load config dynamically in scripts.

Exercise 15: Automated Daily Data Workflow

  - Fetches data automatically
